# 结构体

PS：与[C语言结构体](../C语言/9.结构体.md "C语言结构体")一样，但不太一样。直接当成两个东西使用吧

C语言结构体的特点：

1. 只能有 **数据**，不能有函数
2. 不能有构造 / 析构
3. 不能继承
4. 不能访问控制



## 定义

直接用`.`成员运算符访问，不需要要 `->`，也不需要`(*p).`，简化了访问

```c++
struct Person {            // 简单记忆法： struct Person = class
    int age;
    char name[20];
};

int main()
{
    Person p;              // C++ 中可以省略 struct
    p.age = 18;            // 直接用.成员运算符访问，不需要要 ->，也不需要(*p).，简化了访问
}
```



## 1.`C++ struct` 可以有「函数」

```c++
#include <iostream>
using namespace std;

struct Person {
    int age;

    void sayHello() {
        cout << "age = " << age << endl;
    }
};

int main()
{
    Person p;
    p.age = 20;
    p.sayHello();
}
```



### 结构体数组

```c++
struct Person {
    int age;
    std::string name;
};

Person people[3] = {
    {18, "Tom"},
    {20, "Jack"},
    {22, "Lucy"}
};

// const auto& = const Person&
for (const auto& p : people) {
    std::cout << p.name << std::endl;
}
// 每次循环都复制一个 Person，但是可以修改值
for (Person p : people) {
    p.age += 1;   // 不影响原数据
}
```

`const auto&` =「自动推导类型的只读引用」，**只是引用，性能更好**

| 部分    | 含义                 |
| ------- | -------------------- |
| `auto`  | 让编译器自动推导类型 |
| `&`     | 引用，不拷贝         |
| `const` | 只读，不能修改       |



## 2.构造函数

`:` 是“初始化”，`=` 是“先创建再修改”

```c++
struct Person {
    string name;
    int age;

public:
    // Person() = default;
    Person() : name(""), age(18) {};
    Person(string n, int a) : name(""), age(a) {}
};

// 使用构造函数创建
Person p(18);
```

如果没有写任何构造函数，编译器会自动生成默认的构造函数；如果写了“有参构造函数”，编译器**不会再生成默认构造函数**。如果类需要支持“无参构造”，那在自定义构造函数后要显式提供默认构造函数。

> 不建议使用以下写法的构造函数，初始化可能会被赋值两次

```c++
struct Person {
    string name;
    int age;
public:
    Person(string n, int a) {
        name = n;
        age = a;
    }
};
```



## 3.析构函数

**析构函数是对象“死亡”时自动执行的最后一段代码**，主要工作是**清理对象在生命周期内申请的资源**，防止内存泄漏等问题

- **名字**：`~类名`，比如类 `MyClass` 的析构函数是 `~MyClass()`。
- **无参数，无返回值**：一个类只能有一个析构函数，不能被重载。
- **自动调用**：你不需要（绝大多数情况下也不应该）手动调用它，编译器会在对象生命周期结束时自动调用。

```c++
class Base {
    int* data;
public:
    ~Base() {
        delete[] data;      // 释放构造函数中申请的内存
    }
};
```

**虚析构函数**：主要用于执行派生类的析构函数

一般的派生类销毁，只会执行基类的析构函数，定义了**虚析构函数**的派生类销毁也会执行派生类的析构函数

```c++
class Base {
    int* data;
public:
    virtual ~Base() {
        delete[] data;      // 释放构造函数中申请的内存
        std::cout << "Base Destructor" << std::endl;
    }
};
```

```c++
class Derived : public Base {
public:
    ~Derived() { std::cout << "Derived Destructor" << std::endl; }
};

int main() {
    Base* ptr = new Derived();
    delete ptr; 
    /* 输出：
       Derived Destructor
       Base Destructor
    */
    return 0;
}
```

如果没有`virtual`则只会输出`Base Destructor`，`virtual`可以让派生类也能执行析构函数，避免派生类的内存泄漏



## 4.继承

`struct`：默认 **public 继承**

`class`：默认 **private 继承**

```c++
struct Student : Person {
    int score;
};
```



## 5.访问控制

```c++
struct Person {
public:
    int age;
private:
    int secret;
};
```



## 结构体和类

在 C++ 里，`class` 和 `struct` **本质上几乎是一样的**，它们的核心区别主要体现在**默认访问权限**和**默认继承方式**上

1. 默认访问权限不同

   `struct`：默认是 **public**

   `class`：默认是 **private**

   ```c++
   struct A {
       int x;  // 默认 public
   };
   
   class B {
       int x;  // 默认 private
   };
   ```

2. 默认继承方式不同，都是继承默认

   `struct`：默认 **public 继承**

   `class`：默认 **private 继承**

3. 其他方面：其实完全一样
