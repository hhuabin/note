# `abstract class` 抽象类

```java
abstract class Animal {
    abstract void makeSound();   // 抽象方法，必须实现

    void eat() {
        System.out.println("eat");
    }
}
```

1. 子类必须实现所有抽象方法
2. 抽象类可以有构造函数
3. 抽象类不能是 `final`
4. 抽象方法不能是 `private / final / static`；`abstract` 要被重写



## 抽象类的应用

### 1.模板方法模式

```java
abstract class Game {
    void play() {
        init();
        start();
        end();
    }

    abstract void init();
    abstract void start();
    abstract void end();
}
```

子类只关心细节

```java
class Football extends Game {
    void init() { System.out.println("足球初始化"); }
    void start() { System.out.println("开始比赛"); }
    void end() { System.out.println("结束比赛"); }
}
```

### 2.代码复用

```java
abstract class BaseService {
    void connectDB() {
        System.out.println("连接数据库");
    }
}
```



# :star: 抽象类 vs 接口

| 对比         | 抽象类           | 接口                         |
| ------------ | ---------------- | ---------------------------- |
| 是否能有实现 | ✅ 可以           | ✅（default/static）          |
| 成员变量     | ✅ 可以有普通变量 | ❌ 只能 `public static final` |
| 构造函数     | ✅ 有             | ❌ 没有                       |
| 继承         | 单继承           | 多实现                       |
| 适用场景     | 有“共性代码”     | 纯规范                       |
