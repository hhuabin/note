# `Modifiers` 修饰符

## `.not`

`.not` 允许您测试它的反面

```typescript
test('isArray returns true', () => {
    expect(isArray([])).not.toBe(false)
})
```



## `.resolves`

使用`resolves` 解开已完成的 Promise 的值

测试异步代码必须要使用`return`，否则就使用 `async/await`

```typescript
test('resolves to lemon', () => {
    // make sure to add a return statement
    return expect(Promise.resolve('lemon')).resolves.toBe('lemon');
})
```

在测试代码中使用 `async/await`，获取同步的返回值

```typescript
test('resolves to lemon', async () => {
    await expect(Promise.resolve('lemon')).resolves.toBe('lemon');
    await expect(Promise.resolve('lemon')).resolves.not.toBe('octopus');
})
```



## `.rejects`

测试异步代码必须要使用`return`，或者结合使用 `async/await`

```typescript
test('resolves to lemon', () => {
    // make sure to add a return statement
    return expect(Promise.reject('lemon')).rejects.toBe('lemon');
})
```



# `Matchers` 匹配器

常用的匹配器；匹配器有很多不用记住所有的，需要前去官网查询即可

## `toBe(value)`

调用`Object.is` 来比较值，比`===` 严格相等运算符更适合测试

```typescript
test('isArray returns true', () => {
    expect(isArray([])).toBe(true)
})
```

## `.toEqual(value)`

`.toEqual` 来递归地比较对象实例的所有属性值相等。它调用 `Object.is` 来比较原始值，他不会对比 `undefined`

```typescript
expect({ a: undefined }).toEqual({})
```

## `.toStrictEqual(value)`

与 `.toEqual` 的区别

- 具有 `undefined` 属性的键将被检查，例如 `{a: undefined, b: 2}` 将不等于 `{b: 2}`；
- `undefined` 项目将被考虑在内，例如 `[2]` 将不等于 `[2, undefined]`；
- 数组稀疏性将被检查，例如 `[, 1]` 将不等于 `[undefined, 1]`；
- 对象类型将被检查，例如，具有字段 `a` 和 `b` 的类实例将不等于具有字段 `a` 和 `b` 的文字对象。

```typescript
expect({}).toStrictEqual({})         // 测试通过

expect({ a: undefined }).toStrictEqual({})       // 测试不通过
```

## `.toBeNull()`

`.toBeNull()` 与 `.toBe(null)` 相同；检查某事物是否为 `null`

```typescript
expect(null).toBeNull()
```

## `toBeUndefined`

使用 `.toBeUndefined` 来检查变量是否未定义

```typescript
expect(undefined).toBeUndefined()
```

## `.toBeDefined()`

使用 `.toBeDefined` 来检查变量是否未定义

```typescript
const a = ''
expect(a).toBeDefined()
```



## `.toBeTruthy()`

检测真值

```typescript
expect(100).toBeTruthy()
```

## `.toBeFalsy()`

检测假值

```typescript
expect(0).toBeFalsy()
expect('').toBeFalsy()
expect(null).toBeFalsy()
expect(undefined).toBeFalsy()
```

使用 `toBeGreaterThan` 来比较数字或大整数的值

```typescript
expect(11).toBeGreaterThan(10)
```

大于等于

## `.toBeLessThan(number | bigint)`

小于

## `.toBeLessThanOrEqual(number | bigint)`

小于等于



## `.toContain(item)`

当你想要检查数组中是否包含某个项目时，使用 `.toContain`

```typescript
expect([1]).toContain(1)

expect(new Set([1])).toContain(1)
```

## `.toMatch(regexp | string)`

使用 `.toMatch` 来检查字符串是否与**正则表达式匹配**

```typescript
expect('10').toMatch('101')
```

## `.toMatchObject(object)`

部分匹配对象结构，只匹配写的部分

```typescript
const user = {
    id: 1,
    name: 'bin',
    age: 18,
    createdAt: '2026-04-24',
}

// 测试通过
expect(user).toMatchObject({
    id: 1,
    name: 'bin',
})
```

`toMatchObject` 与 `toEqual`

`toMatchObject` 只会检查需要的属性，`toEqual`只会排除 `undefined` 的属性



## `.toThrow(error?)`

使用 `.toThrow` 测试函数在被调用时是否抛出异常

```typescript
const func = () => {
    throw new Error('')
}
expect(func).toThrow()
```

测试不抛出异常

```typescript
const func = () => {}
expect(func).not.toThrow()
```
