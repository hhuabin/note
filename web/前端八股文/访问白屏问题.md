# 访问白屏问题

## 白屏原因

1. JS 资源加载失败；比如`chunk`文件加载失败

   加载失败的原因有：网络错误、资源被删除、版本不一致、CDN异常等

2. 运行时错误

3. 缓存版本不一致导致的



## 解决方法

1. 处理 chunk 加载失败，例如 Loading chunk failed 时自动刷新页面

   同时使用以下三种方案

   ```typescript
   window.addEventListener('error', (event) => {
       const msg = event.message || ''
   
       if (msg.includes('Loading chunk')) {
           console.log('chunk 加载失败')
           window.location.reload()
       }
   })
   ```

   ```typescript
   window.addEventListener('unhandledrejection', (event) => {
       const reason = event.reason?.message || ''
   
       if (reason.includes('Loading chunk')) {
           console.log('chunk 加载失败')
           window.location.reload()
       }
   })
   ```

   ```typescript
   const Home = React.lazy(() =>
       import('./Home').catch(() => {
           window.location.reload()
       })
   )
   ```

2. 针对运行时错误，使用 `ErrorBoundary` 捕获渲染错误

3. 资源缓存策略

   - HTML 设置 `no-cache`，`JS/CSS` 使用 `hash` 并设置长缓存
   - 发布新版本时候，服务器不删除历史版本资源，只在原来的基础上添加
   - 版本对比（本地 vs 远程），本地与远程版本不一致时，触发 `window.location.reload()`

4. 第四是做白屏监控，通过检测 `root` 节点是否渲染成功来上报异常

   ```typescript
   setTimeout(() => {
       const root = document.getElementById('root')
   
       if (!root || root.children.length === 0) {
           // 异常上报
           reportWhiteScreen()
       }
   }, 10000)
   ```
