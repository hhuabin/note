# `iframe` 的问题

1. 页面切换体验差

    `iframe` 本质上是一个独立网页，点击菜单时，会造成白屏、重新请求资源，体验不如 SPA

    ```
    iframe A 卸载
    iframe B 加载
    ```

2. 通信困难

    ```javascript
    // 主应用
    iframe.contentWindow.postMessage(data)
    
    // 子应用
    window.addEventListener('message', fn)
    ```

    业务多了以后，消息满天飞。

3. 路由难统一

    刷新页面时，在相同的路由下，`iframe` 可能会回到首页

4. 资源重复加载

    三个 `iframe` 都是 `Vue` 的话，会加载三次 `Vue.js`，并且每个`iframe`都有独立的 `window` 和上下文 `Document`



# `qiankun`

`qiankun` 解决的问题

1.   多团队并行开发
2.   技术栈共存（Vue2、Vue3、React）
3.   SPA 体验



## `qiankun`基本原理

`iframe` 标签会创建一个**全新的、独立的**浏览器上下文（新的 `window`、`document`、`history`）

区别于`iframe`，**`qiankun` 的本质是==从其他端口下载 JS==，然后在沙箱中执行**，在**主应用同一个页面环境**中运行，通过沙箱模拟隔离
