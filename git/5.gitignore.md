# 忽略规则

### 1. 忽略特定文件（当前路径）

```bash
# 忽略特定文件
filename.txt
```

### 2. 忽略特定类型的文件

```bash
# 忽略所有 .log 文件
*.log
```

### 3. 忽略特定文件夹

```bash
# 忽略特定文件夹
foldername/

# 忽略所有层级的文件夹
**/node_modules/
```

### 4. 忽略特定路径下的所有内容

```bash
# 忽略特定路径下的所有内容
path/to/directory/
```

### 5. 使用通配符

```shell
# 使用通配符来忽略所有 .pdf 文件
*.pdf

# 忽略以 temp 开头的文件或文件夹
temp*/
```

### 6. 排除部分文件或文件夹

```bash
# 忽略所有 .txt 文件，但不忽略特定文件
*.txt
!importantfile.txt
```

### 7. 注释

```bash
shellCopy code
# 这是注释 - 以 # 开头的行会被视为注释
```

### 8. 忽略空文件夹

Git 默认不会跟踪空文件夹，所以通常不需要显式地将空文件夹加入 `.gitignore`。



## 移除版本控制

1.   **从版本控制中移除 `folder` 文件夹：** 运行以下命令将 `folder` 文件夹从版本控制中移除，但不会删除实际的文件

     ```bash
     git rm -r --cached folder
     
     git reset         # 通过 git reset 回退
     ```

2.   强制重新添加文件（如果文件已被忽略）

     ```bash
     git add -f <file>
     ```

     - `-f` 或 `--force`：强制添加文件，即使它被 `.gitignore` 忽略

3.   移除 `aaa.js` 文件的 `git` 追踪；该命令一定要是文件，不能是文件夹

     ```bash
     git update-index --assume-unchanged src/aaa.js
     ```

4.    列出所有被忽略的文件

     ```bash
     git status --ignored
     ```

     



## `LF`、`CRLF`换行符问题

`Windows` 的默认换行符为 `CRLF`，`mac/Linux` 的默认换行符为 `LF`

```bash
# 查看本地文件的换行符类型
file aaa.js
```

-   输出示例
    -   `ASCII text, with CRLF line terminators` → 有`CRLF `，Windows 风格（`CRLF`）
    -   其他为`LF`



### :star: 1.最简单的适配换行符方式

查看`core.autocrlf`（自动转换`crlf`）的值（用户级别的全局配置）

```bash
git config --get core.autocrlf
```

`core.autocrlf`的值有三个

```bash
true
false
input
```

权限：**系统级 → 用户级（global）→ 项目级（local）(仓库里的`.git/config`可能单独写了 `autocrlf = false`)**；

项目可以自定义`core.autocrlf`，所以，下面的不一定生效，保险还是用方法2

1.   ​	`true`

     ```bash
     # Windows 配置
     git config --global core.autocrlf true
     ```

     ```
     提交时：
     CRLF -> LF
     
     检出时：
     LF -> CRLF
     ```

2.   `false`

     ```bash
     # 通常不配置
     git config --global core.autocrlf false
     ```

     ```
     提交时：
     不转换
     
     检出时：
     不转换
     ```

3.   `input`

     ```bash
     # Mac/Linux 配置
     git config --global core.autocrlf input
     ```

     ```
     提交时：
     CRLF -> LF
     
     检出时：
     不转换
     ```



### 2.自定义配置`.gitattributes`(`.gitattributes`的优先级比`core.autocrlf`高)

创建`.gitattributes`文件，定义拉取 / 提交时统一行结尾为 `LF`

```bash
# 根据 .gitattributes 的规则，修改当前目录及其所有子目录的所有文件，并放到暂存区
git add --renormalize ./
```

```bash
# .gitattributes 文件设置，统一 Git 行为

# ------------------------------
# 通用文本处理，统一行结尾为 LF
# ------------------------------
* text=auto
* text eol=lf

# ------------------------------
# 前端源码，拉取 / 提交时统一行结尾为 LF
# ------------------------------
*.js      text eol=lf
*.jsx     text eol=lf
*.ts      text eol=lf
*.tsx     text eol=lf
*.vue     text eol=lf

*.css     text eol=lf
*.scss    text eol=lf
*.less    text eol=lf

*.html    text eol=lf
*.json    text eol=lf
*.yaml    text eol=lf
*.yml     text eol=lf
*.md      text eol=lf

*.svg     text eol=lf

# ------------------------------
# 二进制文件，不转换
# ------------------------------
*.png     binary
*.jpg     binary
*.jpeg    binary
*.gif     binary
*.ico     binary
*.webp    binary
```

也可以配合`.editorconfig`统一编辑器行为（`vscode`、`idea`都通用）

-   `idea`的话，也可以在设置的`Code Style`配置
-   `vscode`则可以在设置的`files.eol`配置

```bash
# .editorconfig 编辑器设置，统一编辑器行为

# 这里是项目根目录，不是则不设置即可
root = true

[*]
# UTF-8 编码
charset = utf-8
# 行结尾为 LF
end_of_line = lf
# 文件结尾添加空行
insert_final_newline = true
# 使用空格缩进
indent_style = space

# 特殊文件的行结尾为 CRLF
[*.bat]
end_of_line = crlf

[*.cmd]
end_of_line = crlf

[*.ps1]
end_of_line = crlf
```



# 注意事项：

- 每行规则描述一个忽略规则，可以是文件、文件夹、通配符等。
- 在编写 `.gitignore` 文件时，可以使用绝对路径或相对路径。
- `.gitignore` 文件也可以位于项目的子文件夹中，**其规则将适用于该子文件夹及其子目录**。
- 忽略规则支持使用通配符、正则表达式和逻辑操作符等，可以实现更精细的文件排除。

在项目中创建一个 `.gitignore` 文件并添加适当的规则，可以确保不需要跟踪的文件或文件夹不会被提交到版本控制系统中。
