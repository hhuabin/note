# `cherry-pick`复制提交内容

核心：只合并需要的 `commit` 提交，`cherry-pick` 复制的是提交内容

如：当前在`master`分支，需要从`dev`分支合并 `a1b2c3d` 提交到 `master`，该情况下可以使用 `cherry-pick`

结果：从 `dev` 复制一份镜像提交到 `master` 分支。最后`master`和`dev`代码可能应该是一样的，但是提交的`ID`肯定是不一样的，如果。`dev` 的提交`ID`是`D`，那么 `master` `cherry-pick`之后的`ID` 应该是 `D'`

1.   合并一个 `commmit`

     ```bash
     git checkout master
     
     # 查看 dev 的 commit
     git log dev
     git log --oneline dev
     
     # 合并需要的内容到 master
     git cherry-pick a1b2c3d
     ```

     ```bash
     master
       A
       B
       C
        \
     dev D
     
        ||
     变成下面的
        ||
     
     master
       A
       B
       C
       D'     # D' 是复制到 master 的新 commit
        \
     dev D
     ```

     commit 内容一样；commit ID 不一样

2.   只合并，不提交

     ```bash
     git cherry-pick -n a1b2c3d
     
     # 或者
     git cherry-pick --no-commit a1b2c3d
     ```

3.   合并多个 `ID`

     ```bash
     git cherry-pick commit1 commit2
     
     # 合并顺序，先合并 commit1 再合并 commit2
     # 所以建议按照提交时间先再到后合并，把先提交的放到前面去
     
     # 如：有三个提交
     # aaa
     # bbb
     # ccc
     
     git cherry-pick ccc bbb aaa
     ```

4.   合并多个连续的提交

     ```bash
     # 记得 ccc 在前头；aaa 在前面的话，大概率什么都合并不到
     git cherry-pick ccc^..aaa
     
     # aaa   新提交
     # bbb
     # ccc   旧提交
     
     # 等价于，前提是这三个提交必须是连续的
     git cherry-pick ccc bbb aaa
     ```



# `revert`

`revert` 相当于 `cherry-pick` 的反操作

-   `cherry-pick` 是复制某一个 `commit` 的提交内容，然后产生一个新的提交记录
-   `revert` 是撤销某一个 `commit` 的提交内容，然后产生一个新的提交记录



创建一个新的 Commit，撤销指定 Commit 的修改

-   **不是删除 Commit**
-   **不是修改历史**
-   **而是新增一个 Commit 来抵消以前的修改**

例如：

```bash
A ---- B ---- C ---- D (HEAD)
```

假设：

```bash
A 创建项目
B 登录功能
C 修改按钮颜色
D 新增订单功能
```

现在发现 C 有问题。

执行：

```bash
git revert C
```

变成：

```bash
A ---- B ---- C ---- D ---- E
```

其中

```bash
E = 撤销 C
```

历史仍然完整。



## 基本语法

1.   撤销某一个提交的内容，然后自动`commit`一个新的记录

     ```bash
     git revert <commit>            # git revert a1b2c3d
     ```

2.   撤销某一个提交的内容，不立即 `Commit`

     ```bash
     git revert --no-commit commit
     ```

3.   撤销多条提交记录

     撤销多条记录应该按照从新到旧的顺序撤销

     ```bash
     A
     B
     C
     D
     E
     
     撤销
     D、C
     
     git revert D
     git revert C
     ```

4.   继续处理冲突

     ```bash
     git revert --continue
     ```

5.   放弃当前 revert

     ```bash
     git revert --abort
     ```

6.   撤销 Merge Commit

     ```bash
     git revert -m 1 <merge-commit>
     ```
