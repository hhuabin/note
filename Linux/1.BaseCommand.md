# 常用命令

### `shutdown`

```shell
shutdown         # 关机，会等一会再关机
shutdown -c      # 取消关机
shutdown -r      # 重启

ifconfig         # 查看计算机当前的网卡配置信息
ping ip地址      # 检测目标主机是否工作正常
```



### `ls` 查看

```shell
ls               # 显示的是当前目录里的所有文件和文件夹
ls -a            # 列出目录所有文件，包含以.开始的隐藏文件
ls -t            # 以文件修改时间排序
ls -l            # 除了文件名之外，还将文件的权限、所有者、文件大小等信息详细列出来
ls -h            # 配合 -l 使用，显示文件大小
```

```shell
ls -lhrt
```



### `tree` 树状结构显示

```shell
tree           # 以树状结构显示目录和文件
tree -L 1      # 显示层级深度，如这里显示层级1
tree -a        # 显示隐藏文件
tree -d        # 只显示目录

tree /a/b/c    # 查看指定目录
```



### `cd` 进入

```shell
cd            # 切换到当前用户的主目录
cd ~          # 切换到当前用户的主目录
cd .          # 保持当前目录不变
cd ..         # 切换到上级目录
cd -          # 在近两次目录中来回切换
```



### `pwd` 查看目录

```shell
pwd           # 查看当前所在目录
```



### `rm` 删除

```shell
rm -r           # 将删除某个目录以及其中所有的文件和子目录
rm -f           # 强制删除，忽略提示
rm -i           # 进行交互式删除，提示确认删除

rmdir directory           # 删除 directory 目录，前提是空的
rmdir -p a/b/c            # 如果 a/b/c 都为空，则删除
```



### `touch` 新建文件

如果文件不存在，创建空白文件；如果文件已存在，修改末次修改日期

```shell
touch test.txt
touch a.txt b.txt c.txt   # 创建多个文件
```



### `mkdir` 新建文件夹

```shell
mkdir directory      # 创建 directory/ 文件夹
mkdir -p a/b/c       # 创建多级文件夹
```



### `cp` 复制文件

```shell
cp a.txt b.txt           # 把 a.txt 复制成 b.txt，如果 b.txt 已存在，则会被覆盖
cp a.txt b.txt demo/     # 复制多个文件到目录

cp -i           # 如果文件将覆盖目标中的文件，他会提示确认
cp -r           # 这个选项会复制整个目录树、子目录以及其他
cp -v           # 显示文件的复制进度
cp -a           # 保留文件属性，如权限、时间等
```



### `mv` 移动

```shell
mv a.txt demo/       # 移动文件
mv a.txt b.txt       # 重命名文件
mv old_dir new_dir   # 重命名目录

mv -i                # 覆盖前询问
mv -a                # 显示移动过程
mv -f                # 强制移动，不询问
mv -n                # 不覆盖已存在的文件
mv -v                # 显示文件的移动进度

mv -n file1.txt /home/user/documents/
```



### `cat` 查看文件内容

```shell
cat text.txt               # 查看文件内容
cat -n test.txt            # 对输出的行进行编号
cat -b test.txt            # 只给非空行编号

cat a.txt b.txt > c.txt    # 拼接文件；把 a+b 合并到 c
cat app.log                # 查看日志
```



### `grep` 文本搜索工具

```shell
grep hello test.txt       # 在 test.txt 中搜索 hello

grep -i          # 忽略大小写
grep -n          # 显示行号
grep -r          # 递归搜索目录（超常用）
grep -v          # 显示不匹配内容；排除匹配的行
grep "^root" /etc/passwd        # 正则搜索

grep --color=auto aaaa          # 高亮显示匹配到的信息
```



### `alias` 别名命令

```shell
# 查看别名
alias                # 查看所有别名
alias ll             # 查看某一个别名

# 创建别名
alias 名称='命令'      # 如 alias ll='ls -al'

# 删除别名
unalias ll           # 删除别名
unalias ll la        # 删除多个别名
unalias -a           # 删除全部别名
```



### `|` 管道符

把“前一个命令的输出”，作为“后一个命令的输入”

```shell
cat text.txt | grep hello          # 查询使用

ps aux | grep nginx

tail -100 app.log | grep ERROR     # 查询日志
```



### `>` 覆盖写入

```shell
echo hello > a.txt             # 把 hello 写入 a.txt，如果不存在则创建
echo world > a.txt

# a.txt 文件的内容为 world
```



### `>>` 追加写入

```shell
echo hello >> a.txt            # 在末尾追加内容
echo world >> a.txt

# a.txt 文件的内容为
# hello
# world
```



## `tail`

`tail` 是 Linux 里专门用来看“文件尾部”的命令

`cat` 需要一次性全部输出，而 `tail` 只输出文件末尾的部分即可

```shell
tail app.log                    # 查看文件最后10行
tail -100 app.log               # 查看最后 100 行

tail -f app.log                 # 实时监控文件新增内容，特别是日志；Ctrl + C 停止；-f 是 tail 的核心，常用于监控日志；
tail -f app.log | grep ERROR    # 筛选需要的日志
tail -F app.log                 # 可以能自动跟上新文件，如文件名被修改，也可以跟踪。更适用于生产环境

# 查看中间部分
tail -1000 app.log | head -100  # 先取文件的最后 1000 行，再过滤取 前面的 100 行，这样就可以达到查看中间部分的效果

tail -f a.log b.log             # 实时查看多个日志
```

## 进程信息

```shell
ps aux               # 查看进程详细状况
```

- a：显示 **所有用户** 的进程
- u：以“用户友好格式”显示详细信息
- x：显示 **没有控制终端** 的进程

```shell
kill PID(进程号)           # 终止进程
kill -9 PID(进程号)        # 强制终止进程
```



## 软硬连接

1. 查看当前目录下的软连接

   ```shell
   ls -l         # 软连接会显示 lrwxrwxrwx 1 root root 5 May 18 c.txt -> a.txt; c.txt 指向了 a.txt
   ```

2. 创建硬连接

   `ln` 原文件 硬连接名

   ```shell
   ln a.txt b.txt          # a, b其实是同一个文件 修改任意一个，另一个也会变
   ```

3. 创建软连接

   `ln -s` 原文件 软连接名

   ```shell
   ln -s a.txt c.txt       # 类似快捷方式，c.txt 里面存的是 a.txt 的路径
   ```

##### 软连接最常见的应用，如本版切换

```
nginx -> nginx-1.28
```

升级时：

```
nginx -> nginx-1.29
```

无需改配置
